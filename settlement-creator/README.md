# 決済案内書 自動作成＆清算監査システム

不動産売買の最終ステップ「決済（引き渡し）」で最もミスが起きやすい
**固都税・管理費等の日割り清算**と**買主用・売主用 決済案内書**の作成を自動化する Streamlit アプリ。

「重要事項説明書PDF」と「固定資産税 評価証明書PDF」の2点をアップロードするだけで、
売買条件・税額・宛名を自動パースし、1円単位で清算した決済案内書（Excel・2シート）を生成します。

## 特長

- **2点アップロード型**：重説PDF＋評価証明書PDFから売買代金・手付金・管理費・税額・氏名を自動抽出
- **固都税の日割り（365日ベース）**：起算日を4月1日（既定）／1月1日で切替。年またぎの期間判定も自動
- **管理費等の先出し清算**：当月日割り＋任意で翌月1ヶ月分を前払い算入
- **月末タイムラグ自動文言**：決済日が20日以降なら売主案内書に引落停止リスクの注意書きを自動挿入
- **買主⇄売主の符号反転**：単一の計算結果から、買主＝持参金（全プラス）／売主＝手取り（経費を天引き）を自動生成
- **外部有料API不使用**：PDF解析は `claude` CLI（Claude Pro/Maxサブスク）で完結。APIキー不要

## セットアップ

```bash
cd settlement-creator
python3 -m venv .venv
./.venv/bin/pip install -r requirements.txt
./.venv/bin/streamlit run app.py --server.port 8510
```

ブラウザで http://localhost:8510 を開く。

PDF解析には Claude Code CLI（`claude` コマンド）が PATH 上にあり、Pro/Max サブスクで
ログイン済みである必要があります。

## 使い方

1. **書類アップロード**：重説PDF・評価証明書PDFを選び「PDFを自動解析」を押す
2. **売買条件・税額の確認**：AI解析結果を確認し、必要なら手修正
3. **決済条件**：決済日・起算日（関東/関西）・管理費の清算方法・諸費用を設定
4. **清算プレビュー**：買主用・売主用の精算金内訳を確認
5. **ダウンロード**：買主用・売主用の2シートを持つ決済案内書 Excel を出力

## 計算ルール

- **固都税**：年税額 T ＝ 固定資産税相当額 ＋ 都市計画税相当額。買主負担 ＝ `round(T × 買主負担日数 / 365)`。
  売主は起算日〜決済日前日、買主は決済日〜期末を負担（決済日は買主負担）。
- **管理費等**：当月日割り ＝ `round(月額 × (当月末 − 決済日 + 1) / 当月日数)`。翌月前払いは月額満額を加算。
- **売主手取り** ＝ 売買残代金 ＋ 固都税清算金 ＋ 管理費等清算金 − 仲介手数料 − 登記費用（抵当権抹消等）。
- **買主持参金** ＝ 売買残代金 ＋ 固都税清算金 ＋ 管理費等清算金 ＋ 仲介手数料 ＋ 登記費用（所有権移転）。

> 端数処理は四捨五入（実務慣習）。複数年度にまたがる複雑な税額按分や、起算日の地域特例が
> ある場合は、プレビューの内訳テキストを確認のうえ必要に応じて手修正してください。
