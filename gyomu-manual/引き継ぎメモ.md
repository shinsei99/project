# 業務マニュアル 引き継ぎメモ（メインPC用）

最終更新: 2026-07-10 / 副機で作成 → Dropbox経由でメインPCへ引き継ぎ

---

## これは何？

「社内・総務/１ヵ月の業務の流れ・引き継ぎ.xls」と各シートの台帳を、
**業務(部門)ごとに分割したWebマニュアル**に再構成したもの。

- **成果物（実物）**: `（★必読★）新共有フォルダ/業務マニュアル.html`
  - ダブルクリックでブラウザ表示。共有フォルダ直下なので全員が開ける。
  - 6部門（全体像カレンダー / 経理 / 管理 / 営業 / 設備 / 総務）に22マニュアル
    ＋**物件別マニュアル索引から開く埋め込みページ31本**（物件別20＋共通11）。
  - 各手順の末尾「関連ファイル・フォルダ」から共有フォルダの実ファイル（申込書・精算明細・鍵貸出管理簿など）へ直リンク。
  - **物件別・共通マニュアル(docx/doc)は「リンク」ではなくHTML内に全文を埋め込んだ「読めるページ」化**（2026-07-11）。
    「物件別マニュアル索引」でカードを選ぶと、ファイルを開かずその場で全文表示。生成時に `textutil`（macOS標準）で
    docx/doc→本文抽出しているので、**再生成はmacOSで実行すること**（＝メインPCでもmacなのでOK）。原本docxが更新されたら
    `python3 generate.py` で作り直せば反映される。閲覧時はファイル不要（HTML自己完結）。検針表・台帳(xlsx)はリンクのまま。
  - **ほぼ全ページを画面上で編集可**（2026-07-11）。「💾保存」はそのブラウザに保存（localStorage）、「↺元に戻す」で配布時に戻す。
    「⧉ generate.py用に書き出し」でコピーしたコードを `generate.py` 冒頭に貼り、メインPCで `python3 generate.py` すると**全員に反映**。
    ページ種別で編集方式と貼り先が異なる：
    - **物件別／共通マニュアル（全文ページ）** … テキスト編集。書き出し先は `OVERRIDES = {}`（キー=docid 例 `bldg-1-man`）。原本docxより優先。
    - **手順書ページ（経理・管理・営業・設備・総務の①②③型）** … 本文まるごと編集（contenteditable）。書き出し先は `HTML_OVERRIDES = {}`（キー=ページid 例 `keiri-nyukin`）。そのページ本文が丸ごと差し替わる。
    - **カレンダー** … 既存の専用エディタ（`CAL = […]` に貼る）。索引ページは編集対象外。
  - 検索窓・印刷ボタン・ダーク/ライト対応。
- **生成スクリプト**: このフォルダの `generate.py`（← 中身を直せばHTMLを作り直せる）

## メインPCで編集・再生成する手順

1. マニュアルの文言・手順・リンクを直したい → `generate.py` を開く
   - 各マニュアルは `manual("id", "部門", "タイトル", "説明", [手順...], [関連リンク...])` の形で並んでいる。
   - 手順は `{"type":"steps","items":[...]}`、台帳は `{"type":"table",...}`。
   - リンクは `L("表示名", "共有フォルダからの相対パス")`。
2. ターミナルで再生成:
   ```
   cd "（★必読★）新共有フォルダ/_業務マニュアル_生成用"
   python3 generate.py
   ```
   → `業務マニュアル.html` が上書き更新される（追加ライブラリ不要・python3のみ）。
3. ブラウザで `業務マニュアル.html` を開いて（⌘R で再読み込み）確認。

※ `generate.py` 冒頭の `OUT =` が出力先パス。パスが変わったらここを直す。

## 補足

- `全ファイル一覧.xlsx`（5シート=5部門のファイル索引）を基にリンクを張った。
  ただしこれは棚卸しスナップショットで、実フォルダは再編後のため一部パスがズレる。
  **リンクは必ず実在する現行フォルダに張ること**（generate.py のパスは実体で確認済み）。
- 元データ: `社内・総務/１ヵ月の業務の流れ・引き継ぎ.xls`（6シート）と
  `社内・総務/研修・資料/マニュアル/`（物件別docx 40本超）。
